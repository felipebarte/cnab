# Status: done
# Dependencies: 10

# Description: Criar parser que processa registros de 240 caracteres com identificação de tipos e segmentos
# Details:
Implementar:
- Função para processar linhas de 240 caracteres
- Identificação de tipos de registro (0=Header Arquivo, 1=Header Lote, 3=Detalhe, 5=Trailer Lote, 9=Trailer Arquivo)
- Identificação de segmentos para registros de detalhe (A, B, J, O)
- Validação básica de estrutura
- Extração de campos comuns (banco, lote, sequencial)

Diferente do CNAB 400 que usa posições fixas simples, CNAB 240 tem estrutura mais complexa com segmentos.

# Test Strategy:
Testar com arquivos CNAB 240 de exemplo, verificar identificação correta de tipos e segmentos
